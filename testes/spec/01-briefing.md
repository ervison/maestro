# Briefing do Projeto

## Contexto
Aplicação web para cadastro de raças de gatos, com foco em um frontend bonito e agradável de usar.

## Objetivos
- Criar uma interface web visualmente atrativa e responsiva.
- Permitir o cadastro e a gestão de raças de gatos. [HYPOTHESIS]
- Facilitar a consulta dos registros cadastrados. [HYPOTHESIS]

## Stakeholders
- Solicitante / dono do produto.
- Usuários finais que irão cadastrar ou consultar raças. [HYPOTHESIS]
- Equipe de desenvolvimento e design. [HYPOTHESIS]

## Escopo

### Dentro do escopo
- Frontend da aplicação.
- Tela(s) de cadastro e listagem de raças. [HYPOTHESIS]
- Design intuitivo, bonito e responsivo. [HYPOTHESIS]

### Fora do escopo / não especificado
- Backend, banco de dados e autenticação. [GAP]
- Campos do cadastro. [GAP]
- Regras de validação. [GAP]
- Integrações externas. [GAP]
- Identidade visual, cores e referências de estilo. [GAP]

## Observação
[HYPOTHESIS] Assumi que “cadastro” inclui pelo menos registrar e consultar raças.
