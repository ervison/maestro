## Lacunas e perguntas em aberto

- [GAP] Qual é o escopo exato da aplicação: apenas frontend demonstrativo ou aplicação completa com backend e banco de dados?
- [GAP] O cadastro de raças deve permitir quais operações: criar, listar, editar, excluir e visualizar detalhes?
- [GAP] Quais campos cada raça de gato deve ter no cadastro? Ex.: nome, origem, porte, peso, expectativa de vida, temperamento, pelagem, cor, descrição, imagem.
- [GAP] Há campos obrigatórios e opcionais? Se sim, quais?
- [GAP] O sistema deve aceitar upload de imagens das raças? Se aceitar, qual formato e tamanho máximo?
- [GAP] Deve existir busca, filtro e ordenação das raças? Quais critérios?
- [GAP] É necessário cadastro de categorias, tags ou características adicionais?
- [GAP] As informações cadastradas precisam ser persistidas em banco de dados ou podem ficar apenas em memória/local storage?
- [GAP] Qual tecnologia de frontend deve ser usada, se houver preferência? Ex.: React, Vue, Angular, Next.js, HTML/CSS/JS puro.
- [GAP] Há preferência por algum framework de estilização ou design system? Ex.: Tailwind, Bootstrap, Material UI, CSS customizado.
- [GAP] O que significa “frontend bonito” neste contexto: há referências visuais, cores, tipografia, estilo ou layout desejados?
- [GAP] A aplicação deve ser responsiva para mobile, tablet e desktop?
- [GAP] Há requisitos de acessibilidade? Ex.: contraste, navegação por teclado, leitores de tela.
- [GAP] Em qual idioma a interface deve ser exibida? Apenas português ou multilíngue?
- [GAP] Existe necessidade de autenticação de usuários?
- [GAP] Se houver autenticação, quais perfis e permissões devem existir? Ex.: administrador, editor, visitante.
- [GAP] O sistema precisa de validações específicas para os dados cadastrados?
- [GAP] Devem existir mensagens de erro, sucesso e feedback visual? Em qual tom/estilo?
- [GAP] O cadastro deve incluir importação/exportação de dados? Se sim, em quais formatos?
- [GAP] Há necessidade de paginação ou infinito para a listagem de raças?
- [GAP] O volume esperado de registros é conhecido?
- [GAP] A aplicação deve consumir uma API já existente ou a API também precisa ser desenvolvida?
- [GAP] Existe algum modelo de dados ou esquema já definido para as raças de gatos?
- [GAP] Há requisitos de auditoria, histórico de alterações ou logs?
- [GAP] A aplicação precisa de testes automatizados? Se sim, quais tipos?
- [GAP] Existe prazo, prioridade ou restrição de entrega que impacte a solução?
- [GAP] Há restrições de hospedagem, deploy ou ambiente de execução?
- [GAP] Existe alguma regra legal ou de conteúdo para textos e imagens usados no cadastro?
