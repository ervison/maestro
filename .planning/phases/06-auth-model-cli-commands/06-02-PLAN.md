# Phase 06 Plan 02: Gap Closure — Delegated UAT Executor Availability

## Goal

Close the UAT blocker where delegated `uat-test-execution` (`gsd-uat-executor`) cannot be invoked, so `/gsd-verify-work` can run delegated automatable checks end-to-end.

## Diagnosed Root Cause

The runtime/tooling environment does not expose a callable delegation interface:

- No `gsd-uat-executor` binary available in PATH
- No generic subagent invocation tool exposed to this session
- `EXECUTION_ROOT` is not set, so required delegated workdir binding is undefined

## Execution Tasks

### Task 1 — Provide delegated executor entrypoint

1. Expose `gsd-uat-executor` as a runnable command in the execution environment **or** expose an equivalent supported Task/subagent invocation API.
2. Ensure invocation contract includes:
   - UAT file path argument
   - Mandatory full UAT file pre-read step
   - Workdir override set to `EXECUTION_ROOT`

### Task 2 — Define and enforce execution root

1. Define `EXECUTION_ROOT` for verify-work sessions.
2. Add pre-flight validation: fail fast with actionable error if `EXECUTION_ROOT` is missing.

### Task 3 — Add verify-work delegation preflight gate

1. Update verify-work bootstrap to check delegated executor availability before attempting delegation.
2. If unavailable, write a structured blocker into UAT and stop delegation branch cleanly.

### Task 4 — Add regression coverage

1. Add workflow-level test that validates delegation path can be invoked and captures delegated results.
2. Add negative-path test asserting clear blocker message when delegation binary/API is absent.

## Verification Commands

Run all after implementation:

```bash
which gsd-uat-executor
printenv EXECUTION_ROOT
gsd-uat-executor --help
gsd-verify-work 6
```

Expected:

- `which gsd-uat-executor` resolves to installed executable
- `EXECUTION_ROOT` is non-empty
- delegated executor help renders without error
- `gsd-verify-work 6` records delegated test execution evidence in `06-UAT.md`

## Non-Goals

- No changes to Phase 06 feature code (`maestro/cli.py`, auth/models logic)
- No behavior changes to provider auth implementation
