  <action>
Refactor `_run_agentic_loop` to delegate HTTP streaming to the provider abstraction per D-02:

1. **Change signature** — Remove `tokens: auth.TokenSet` parameter. Add `provider: ProviderPlugin` parameter instead. The provider already handles its own auth.

2. **Add sync wrapper for async provider** — Since `_run_agentic_loop` is synchronous but `provider.stream()` is async, create a helper:
   ```python
   import asyncio
   def _run_provider_stream(provider, messages, model, tools):
       async def _inner():
           result = []
           async for chunk in provider.stream(messages, model, tools):
               result.append(chunk)
           return result
       return asyncio.run(_inner())
   ```

3. **Convert TOOL_SCHEMAS to neutral Tool types** — Create helper to convert raw dicts:
   ```python
   from maestro.providers.base import Message, Tool, ToolCall
   def _convert_tool_schemas(schemas: list[dict]) -> list[Tool]:
       return [Tool(name=s["name"], description=s.get("description", ""), parameters=s.get("parameters", {})) for s in schemas]
   ```

4. **Convert loop messages to neutral Message types** — Replace LangChain BaseMessage usage with neutral Message:
   - `HumanMessage(content=x)` → `Message(role="user", content=x)`
   - `AIMessage(content=x)` → `Message(role="assistant", content=x)`
   - System instructions go in a `Message(role="system", content=instructions)`

5. **Replace httpx.stream() with provider.stream()** — The inner iteration loop:
   - Call `_run_provider_stream(provider, messages, model, tools)`
   - The provider yields `str` chunks (for streaming text) and a final `Message`
   - The final `Message` has `tool_calls: list[ToolCall]` if tools were invoked
   - If `tool_calls` is empty → return `message.content` (final answer)
   - If `tool_calls` is non-empty → execute each tool, append tool results as `Message(role="tool", ...)`, loop again

6. **Auth error surfaces from provider** — The provider's `stream()` already raises `RuntimeError("Not authenticated. Run: maestro auth login chatgpt")` per D-03. Let that propagate unchanged.

7. **Remove direct httpx/SSE imports** — Delete `import httpx` and the ChatGPT-specific imports (`RESPONSES_ENDPOINT`, `_headers`, etc.) from agent.py.

8. **Update `run()` function** — Change to acquire provider via registry per D-01:
   ```python
   from maestro.providers.registry import get_default_provider
   
   def run(...):
       provider = get_default_provider()
       # ... call _run_agentic_loop with provider instead of tokens
   ```
   Per D-03: Do NOT add an auth pre-check in `run()`. The provider raises the actionable unauthenticated error when `stream()` is called, and the loop surfaces it.

9. **Preserve _call_responses_api** — This helper is used by `maestro models --check`. Keep it unchanged for now (it uses direct httpx). Per D-05 (minimal changes), leaving it unchanged is acceptable if tests pass.

Key constraint: The refactor must preserve existing behavior exactly. The provider abstraction is transparent to callers.
