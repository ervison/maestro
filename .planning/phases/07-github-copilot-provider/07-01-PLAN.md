  <name>Task 1: Create CopilotProvider with OAuth device code flow</name>
  <files>maestro/providers/copilot.py</files>
  <action>
Create `maestro/providers/copilot.py` implementing the full CopilotProvider class:

**OAuth Device Code Flow (`login()` method):**
1. POST to `https://github.com/login/device/code` with:
   - `client_id`: `Ov23li8tweQw6odWQebz` (per D-01)
   - `scope`: `read:user`
   - Headers: `Accept: application/json`
2. Extract `device_code`, `user_code`, `interval` from response
3. Print instructions: `Go to: https://github.com/login/device`, `Enter code: {user_code}`
4. Poll `https://github.com/login/oauth/access_token` with:
   - `client_id`, `device_code`, `grant_type`: `urn:ietf:params:oauth:grant-type:device_code`
   - Sleep `interval + 5` seconds between polls (POLLING_SAFETY_MARGIN)
5. Handle responses:
   - `authorization_pending`: continue polling at current interval
   - `slow_down`: increment interval by 5 seconds (AUTH-07), continue polling
   - `expired_token`, `access_denied`: raise RuntimeError
   - `access_token` present: save token and return
6. Store token via `auth.set("github-copilot", {"access_token": token})` — token is `ghu_...` format, long-lived

**stream() method:**
1. Check auth: `auth.get("github-copilot")` — raise RuntimeError if None
2. Convert neutral `Message`/`Tool` to OpenAI chat completions format (NOT Responses API):
   - `{"role": "user", "content": "..."}` for user messages
   - `{"role": "assistant", "content": "...", "tool_calls": [...]}` for assistant
   - `{"role": "tool", "tool_call_id": "...", "content": "..."}` for tool results
   - `{"role": "system", "content": "..."}` for system messages
3. Build request payload:
   ```python
   payload = {
       "model": model,
       "messages": converted_messages,
       "tools": [{"type": "function", "function": {...}} for tool in tools],
       "stream": True,
   }
   ```
4. POST to `https://api.githubcopilot.com/chat/completions` with headers (per D-02):
   - `Authorization: Bearer {token}`
   - `x-initiator: user`
   - `Openai-Intent: conversation-edits`
   - `Content-Type: application/json`
5. Parse SSE stream using `httpx_sse.aconnect_sse`:
   - Yield `str` for text deltas (`choices[0].delta.content`)
   - Buffer `tool_calls` deltas until `finish_reason == "tool_calls"`
   - On `[DONE]`, yield final `Message` with complete content and tool_calls

**Other ProviderPlugin methods:**
- `id`: return `"github-copilot"`
- `name`: return `"GitHub Copilot"`
- `list_models()`: return `["gpt-4o", "gpt-4o-mini", "gpt-4-turbo", "gpt-3.5-turbo"]`
- `auth_required()`: return `True`
- `is_authenticated()`: return `auth.get("github-copilot") is not None`

**Wire format helpers (OpenAI chat completions, NOT Responses API):**
- `_convert_messages_to_wire(messages: list[Message]) -> list[dict]`
- `_convert_tools_to_wire(tools: list[Tool]) -> list[dict]`
- `_parse_tool_call_delta(delta: dict) -> ToolCall`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI → GitHub OAuth | User credentials flow through GitHub's device code endpoint |
| CLI → Copilot API | Bearer token crosses to external API |
| Copilot API → CLI | SSE stream contains LLM responses (untrusted content) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Spoofing | OAuth device code URL | mitigate | Hardcode GitHub URL constants, no user input in URLs |
| T-07-02 | Tampering | Access token storage | mitigate | Use auth.set() which writes with 0o600 permissions |
| T-07-03 | Information Disclosure | Token in logs | mitigate | Never log access_token; logger.debug only for flow state |
| T-07-04 | Denial of Service | Infinite polling | mitigate | 15-minute timeout on device code polling |
| T-07-05 | Elevation of Privilege | Scope creep | accept | read:user scope is minimal; cannot escalate via this token |
</threat_model>
