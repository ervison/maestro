# Phase 4 Plan 01: Config & Provider Registry

**Objective:** Implement runtime provider discovery, config system, and model resolution chain for multi-provider support.

## Context

Phase 3 completed the ChatGPTProvider migration. Phase 4 builds the infrastructure for provider discovery and configuration management. This enables:

1. Runtime discovery of providers via `importlib.metadata` entry points
2. Config file at `~/.maestro/config.json` for default provider/model settings
3. Model resolution following priority chain: `--model` flag → `MAESTRO_MODEL` env → `config.agent.<name>.model` → `config.model` → first authenticated provider
4. Provider registry with `get_provider()` function

## Success Criteria

From ROADMAP.md Phase 4 requirements:

1. `get_provider("chatgpt")` returns the ChatGPT provider instance via `importlib.metadata` entry point discovery
2. `get_provider("nonexistent")` raises `ValueError` with a list of available provider IDs
3. `resolve_model()` follows the priority chain: `--model` flag → `MAESTRO_MODEL` env → `config.agent.<name>.model` → `config.model` → first model of first authenticated provider
4. Model string `"provider_id/model_id"` format is validated; invalid format raises `ValueError` with guidance message
5. Absent `~/.maestro/config.json` falls back gracefully to ChatGPT as default provider

## Tasks

### Task 1: Create Config Module

Create `maestro/config.py` with:
- `Config` dataclass with fields: `model`, `agent` (dict of agent configs)
- `load()` function to read `~/.maestro/config.json`
- `save()` function to write config with 0o600 permissions
- `get()` function to get nested values with dot notation

### Task 2: Create Provider Registry

Create `maestro/providers/registry.py` with:
- `discover_providers()` using `importlib.metadata.entry_points(group="maestro.providers")`
- `@lru_cache(maxsize=1)` to cache discovery results
- `get_provider(provider_id)` that returns provider instance or raises ValueError
- `list_providers()` that returns all discovered provider IDs
- `get_default_provider()` that returns first authenticated provider or "chatgpt" fallback

### Task 3: Create Model Resolution

Create `maestro/models.py` with:
- `resolve_model(model_str: str | None)` following priority chain
- `parse_model_string(model_str: str) -> tuple[str, str]` validating "provider_id/model_id" format
- `get_available_models()` that aggregates models from all authenticated providers
- Proper error messages for invalid formats

### Task 4: Add Tests

Create tests covering:
- Provider discovery via entry points
- `get_provider()` success and failure cases
- Model resolution priority chain
- Config file operations
- Model string parsing validation

### Task 5: Export from Package

Update `maestro/__init__.py` to export:
- `get_provider` from `maestro.providers.registry`
- `resolve_model` from `maestro.models`
- `Config` from `maestro.config`

### Task 6: Update CLI Integration

Modify `maestro/cli.py` to:
- Import and use `resolve_model()` when `--model` flag is provided
- Import and use `get_provider()` to get the provider instance
- Wire provider into agent loop (preparation for Phase 5)
