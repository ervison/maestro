<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| test env → GitHub OAuth | Test sends real HTTP to github.com for device_code and token |
| test env → Copilot API | Test sends real HTTP to api.githubcopilot.com with a live ghu_ token |
| env vars → auth store | MAESTRO_COPILOT_TOKEN is read from env and written to a (temp) auth file |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-01 | Information Disclosure | `MAESTRO_COPILOT_TOKEN` env var | mitigate | Token is written to `tmp_path`-scoped auth file (not `~/.maestro/auth.json`) — monkeypatch auth file path in test; clean up after test completes |
| T-16-02 | Tampering | Test writing to real `~/.maestro/auth.json` | mitigate | Monkeypatch `maestro.auth.AUTH_FILE` to `tmp_path / "auth.json"` so real credentials are never overwritten by test |
| T-16-03 | Elevation of Privilege | Token logged in CI output | accept | ghu_ tokens are long-lived but scoped to Copilot API read access; test output should be reviewed in CI log settings — low risk for this tool's threat model |
| T-16-04 | Denial of Service | Unconstrained polling loop in login() | accept | Existing 15-minute timeout in `login()` is sufficient; smoke test is human-supervised when run in interactive mode |
</threat_model>
