# 260422-t7: SDLC Reflect Loop — Iterative Quality Evaluation

**Goal:** After all 13 artifacts are generated in the `discover` phase, run an iterative LLM-based evaluation loop using a 10-metric rubric. The loop reads all spec files, scores each dimension, diagnoses top problems, applies surgical corrections, and repeats until mean score ≥ 8.0 (or max iterations reached). All corrections happen in-place on the `spec/` files. A final score report is appended to the `DiscoveryResult`.

**Architecture:**
- New file `maestro/sdlc/reflect.py` — contains `ReflectLoop` class and `run_reflect_loop()` entry point
- `harness.py` modified — after the generation loop, if provider is set, call `await run_reflect_loop(...)`
- `schemas.py` modified — add `ReflectReport` dataclass and `ReflectCycle` dataclass; add `reflect_report` field to `DiscoveryResult`
- `cli.py` modified — add `--no-reflect` flag (default: reflect runs when provider is set) and `--reflect-max-cycles` (default: 5)

**Rubric (10 dimensions, 0–10 each, target mean ≥ 8.0):**
1. Cobertura de domínio
2. Consistência de nomenclatura
3. Alinhamento modelo ↔ API
4. Cobertura de RN em ACs
5. Coerência PRD ↔ técnico
6. Qualidade dos ADRs
7. Plano de testes vs. escopo
8. Qualidade individual
9. Rastreabilidade (gaps → decisões)
10. Integridade dos artefatos

### Task 2 — Implement reflect.py

**Files:** `maestro/sdlc/reflect.py` (new)

**Action:** Create `ReflectLoop` class with:

```python
DIMENSIONS = [
    "Cobertura de domínio",
    "Consistência de nomenclatura",
    "Alinhamento modelo ↔ API",
    "Cobertura de RN em ACs",
    "Coerência PRD ↔ técnico",
    "Qualidade dos ADRs",
    "Plano de testes vs. escopo",
    "Qualidade individual",
    "Rastreabilidade (gaps → decisões)",
    "Integridade dos artefatos",
]
TARGET_MEAN = 8.0
```

Key methods:
- `_read_spec_files(spec_dir)` — reads all `*.md` files from spec_dir, returns `dict[filename, content]`
- `_build_eval_prompt(spec_contents)` — builds the STEP 1+2 evaluation prompt (score all 10 dimensions + identify top-3 problems). Expects LLM to respond with structured JSON: `{"scores": [{"dimension": str, "score": float, "justification": str}], "problems": [{"file": str, "dimension": str, "what_to_change": str}]}`
- `_build_fix_prompt(spec_contents, problems)` — builds the STEP 3 correction prompt. Instructs LLM to return a JSON array of file patches: `[{"file": str, "old": str, "new": str}]`
- `_apply_patches(spec_dir, patches)` — applies `old → new` replacements to files on disk; validates that `old` exists in file before replacing (skip + warn if not found)
- `async run(provider, model, spec_dir, max_cycles)` — the main loop:
  1. Read spec files
  2. Call LLM for evaluation (STEP 1+2)
  3. Parse JSON response → build `ReflectCycle`
  4. If mean ≥ 8.0 → stop (passed=True)
  5. Call LLM for corrections (STEP 3)
  6. Parse patches → apply → record corrections in cycle
  7. Repeat from step 1 with updated file contents
  8. On max_cycles reached → stop (passed=False)
  9. Return `ReflectReport`

Error handling:
- If LLM returns malformed JSON, skip cycle and retry (count as a cycle)
- If a patch's `old` string is not found in the file, log warning and skip that patch

**Verify:** `python -c "from maestro.sdlc.reflect import ReflectLoop; print('ok')"`

**Done:** Module importable, `ReflectLoop` class exists with `run()` method.
