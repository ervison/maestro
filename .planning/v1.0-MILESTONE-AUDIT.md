## Gap Fixes Applied (Pre-Audit)

Five integration seams were identified as broken and fixed before finalizing this audit:

| Fix | Requirement | Change |
|-----|-------------|--------|
| AGG-01 | Aggregator system prompt | `aggregator_node()` now sends `AGGREGATOR_SYSTEM_PROMPT` as system message |
| PLAN-03 | Planner imports DOMAINS | `planner/node.py` imports `DOMAINS` from `maestro/domains.py` (no longer hardcoded) |
| AUTH-03 | ChatGPT login routing | `maestro auth login chatgpt` routes through `ChatGPTProvider.login(method)` |
| AUTH-02/06 | Auth status union | `auth status` unions `auth.all_providers()` with discovered providers |
| CONF-02/PLAN-02 | resolve_model usage | Planner and aggregator call `resolve_model(agent_name=...)` instead of removed helpers |
| CONF-01/PLAN-02 | Per-agent model resolution | `run_multi_agent()` no longer injects `--model` into planner/aggregator state; each resolves its own model via `resolve_model(agent_name=...)` honouring `config.agent.<name>.model` |

**Requirement:**  
WORK-06: A Worker can recursively spawn a sub-Planner on its own subtask (optional, depth-guarded)

**Finding:**

**Risk:** None â€” requirement is explicitly marked `(optional)` in the spec. The depth guard infrastructure is in place for when this is implemented.
